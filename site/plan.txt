# План розробки веб-сайту World Gen

## 1. Технологічний стек
- **Backend:** Flask (Python)
- **Frontend:** HTML5, CSS3, JavaScript (Vanilla)
- **Стилізація:** Tailwind CSS
- **3D Рендеринг:** Three.js (WebGL)
- **Комунікація в реальному часі:** WebSockets (через Flask-SocketIO)

## 2. Архітектура сайту (Single Page Application)
Сайт складатиметься лише з однієї сторінки (SPA), яка забезпечить максимально простий та безперервний користувацький досвід.

- **Один екран без перезавантажень:** Всі дії відбуваються на єдиній HTML-сторінці.
- **Цикл взаємодії користувача:**
  1. Користувач налаштовує параметри планети (Subdivisions, Radius, Seed, Warp) у зручній боковій панелі.
  2. Натискає кнопку "Згенерувати" (Generate).
  3. Інтерфейс відправляє запит з параметрами через WebSocket на сервер.
  4. Користувач спостерігає за покроковою анімацією генерації планети в реальному часі на 3D-сцені, яка займає центральну частину екрану.
- **Розподіл обов'язків на фронтенді:**
  - `index.html` — єдиний файл розмітки з підключеним Tailwind CSS.
  - `main.js` — логіка UI (обробка кліків, повзунків) та управління WebSocket-з'єднанням.
  - `scene.js` — інкапсульована логіка Three.js (камера, світло, геометрія, анімація "виростання" рельєфу).

## 3. Архітектурне рішення для анімації генерації в реальному часі
**Найкращий підхід для цього проекту:** WebSocket стрімінг етапів генерації + Three.js BufferGeometry Update.

Оскільки основна логіка (шум, domain warping, клімат) написана на Python, генерувати все на сервері відразу і відправляти готовий результат — це статично і не покаже "процес". Щоб зробити анімацію без переписування всієї логіки на GLSL-шейдери в браузері, ми використаємо покрокову передачу стану:

1. Клієнт відправляє конфіг (seed, radius, subdivisions, тощо) через WebSocket.
2. Flask-бекенд ініціалізує генератор і починає повертати дані **поетапно** (yield або окремі події):
   - *Етап 1 (База):* Створюється базова сферична сітка. Frontend миттєво відмальовує гладку сферу.
   - *Етап 2 (Низькочастотний рельєф):* Застосування базового шуму (наприклад, континентів). Бекенд надсилає масив нових висот (або позицій вершин).
   - *Етап 3 (Деталізація та Warping):* Застосування domain warping та високочастотного шуму. Передаються уточнені висоти.
   - *Етап 4 (Біоми):* Розрахунок температур/вологи та біомів. Бекенд надсилає масив кольорів `(V, 3)`.
3. **Frontend-анімація:** Three.js приймає масиви і за допомогою `requestAnimationFrame` плавно інтерполює (змінює) поточні позиції вершин до нових цільових позицій (ефект "виростання" гір та формування западин). Потім плавно "проявляються" кольори біомів. Обов'язково викликається `geometry.attributes.position.needsUpdate = true`.

*(Примітка: Альтернативою є генерація рельєфу прямо на відеокарті через Custom Shaders у Three.js, що дасть 60 FPS реального часу під час руху повзунків, але тоді Python-код генерації не буде використовуватись. Тому WebSocket-стрімінг — це найкращий шлях поєднати існуючий потужний Python-движок і гарний візуал).*

## 4. Структура інтерфейсу (UI/UX)
- **Бокова панель (Tailwind Sidebar):**
  - **Налаштування:** Subdivisions (повзунок 1-6 для веб-версії, щоб уникнути зависань), Radius, Seed (з кнопкою "Randomize"), Warp Strength.
  - **Керування:** Кнопка "Згенерувати" (Generate).
  - **Експорт:** Кнопки "Завантажити PNG", "Завантажити OBJ", "Завантажити JSON".
  - **Прогрес:** Текстовий або візуальний лог поточного етапу ("Calculating noise...", "Applying biomes...").
- **Основна зона (Three.js Canvas):**
  - 3D-сцена з моделлю планети на весь екран (крім бокової панелі).
  - Орбітальний контроль (мишка: обертання, скрол: зум).
  - Освітлення (Directional Light + Ambient Light, щоб підкреслити тінями рельєф гір).
  - Зоряний фон (Space background) для атмосфери.

## 5. Поетапний план реалізації

**Крок 1: Базовий Flask-сервер та налаштування сайту**
- Створити `app.py` у папці `site/`.
- Налаштувати статичні файли (CSS/JS) та шаблони (HTML).
- Підключити Tailwind CSS та Three.js.

**Крок 2: Верстка інтерфейсу (HTML + Tailwind)**
- Створити сучасну темну тему.
- Зверстати панель управління поверх 3D канвасу (абсолютне позиціонування зліва).

**Крок 3: Інтеграція Three.js на клієнті**
- Ініціалізувати сцену, камеру, світло.
- Написати функцію створення порожньої BufferGeometry, яка буде приймати TypedArrays (Float32Array) для вершин та кольорів.

**Крок 4: WebSockets та бекенд-генерація**
- Додати `Flask-SocketIO`.
- Зробити рефакторинг або написати адаптер для `PlanetGenerator`, який розіб'є функцію `generate_heightmap` та `assign_biomes` на кроки і буде відправляти події `emit('generation_step', data)` клієнту.

**Крок 5: Клієнтська анімація генерації (Morphing)**
- Написати JS-логіку прослуховування WebSocket.
- Реалізувати функцію плавного переходу (lerp) між попереднім масивом висот і новим, щоб планета "оживала" при кожному кроці розрахунків сервера.

**Крок 6: Підключення експортерів**
- Реалізувати REST API ендпоінти у Flask (`/export/obj`, `/export/png`), які отримуватимуть поточні параметри планети, швидко генеруватимуть (або братимуть з кешу) і віддаватимуть файл клієнту.
